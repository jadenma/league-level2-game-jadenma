JTG

by: Jaden Ma

This game was made for my level 2 project at The League of Amazing Programmers.

1. Why I Made This Game

I decided to make this game, because I wanted a unique, multiplayer game, because multiplayer games are more fun, for me at least. I also wanted to make a game where you could shoot bullets, and both players can both be playing at once. This was also not as complicated to make or understand.

2. What I Learned

I learned that minor weird things about your game may not matter to you, but others will find them weird. Some feedback that I got included adding a lot of new functions to my game that I originally did not think of adding.

3. Architecture of My Game

There are 3 pages of my game - the menu, game, and end page. The menu has instructions on how to play, the game is where you play, and the end page is the results of who won, and it will direct you back to the menu. My game has 9 classes - Main, GamePanel, GameObject, ObjectManager, Tank, Tank2, Projectile, Projectile2, and Font. The Main class has the main method, which runs all the code, and it also sets up the game's frame and panel for the people to play on. GamePanel has code for what to put on the panel; for example, loading images, updating the location of tanks and projectiles, when to switch to the 3 pages and what the 3 pages show, managing the tank ammo/lives, and also seeing which keys you press. GameObject is a super class for objects. The objects in the game (tanks and projectiles) inherit the variables in that class. ObjectManager creates the tanks and the projectiles, and it tells whether a bullet hits a tank. The tank classes shows the tank image and manages the speed and direction. The projectile classes do the same thing but with the projectiles, and the font class makes the fonts and font sizes for the text on the 3 different pages.

4. Technical Implementation

In the game, two tanks face each other and try to shoot each other with projectiles while moving around the map and dodging the other person. Each player gets 3 lives and loses a life each time someone is hit. They also start with 5 ammo, and cannot shoot without ammo. Ammo regenerates at 1 ammo per second, and it caps at 5. After all 3 lives of one player are lost, the game ends. One part of the game that I am proud of is how the ammo works. I made it so after a person presses the key to shoot, they lose 1 ammo, and I made a timer for 1 second, and every time the timer happened/went off, 1 ammo would regenerate if the ammo was not 5 or above.

5. Most Challenging Part

The most challenging part of the project was making the movement of the tanks clean, because when I first wrote the code the movement was very rough, and it would move at a few pixels at a time, so I fixed it by adding 2 new speed variables to GameObject. When I made the movement smooth, the tank would go a little bit out of bounds, and then jump back, because my code made it so that if the tank went out of bounds, it would go back to the edge of the in bound area. I fixed it by making it so that once it hit a slightly earlier point, it would go to the point on the edge of the boundary. When I stopped it from jumping back, it would get stuck on the edge, which was just an error in my code, so I fixed those errors and the movement was fixed.

6. Future Fixes and Updates

If I were to continue working on this game, I probably would make it so that if the tanks both got hit at the same time or within a fraction of a second, they would draw, because right now it is slightly favored toward tank1, which is the player on the left. Also, one other small bug that I found is that if a tank is on the very edge of one of the boundaries, they can only move in the perpendicular direction of edge that they are on - the tank won't move parallel/along the edge if you are directly on it. Some improvements or other things I could add to my game could be a timer on the game, so if the game went on for too long they would just tie, or whoever has more lives as the timer goes off wins. I could also add extra functions, like maybe an ability for each tank that could make the game more fun and interesting, like an ultimate/super that could buff/boost a tank for a certain duration.
